# 勤怠アプリ ⇄ スプレッドシート連携 セットアップ手順

勤怠アプリの「シートに同期」ボタンを押すと、Google スプレッドシートに記録が自動で書き込まれます。

- **明細シート**：1日に何回でも・複数回バラけた勤務を、打刻そのままで全部記録
- **月次シート**：Stoke公式フォーマットに、日別集計を自動入力
  - 出社=その日の最初／退社=その日の最後／休憩=空き時間合計／**勤務時間=実働合計**／深夜労働／**深夜休憩時間**
  - 普通残業・40H超残業・休日出勤の列は **既存の計算式のまま** です（上書きしません）。「勤務時間」が入るとテンプレ側の式で自動計算されます

---

## 1. スプレッドシート側に Apps Script を設置

1. 連携したいスプレッドシートを開く
2. メニュー **拡張機能 → Apps Script** を開く
3. 既存の `Code.gs` の中身を全部消し、本フォルダの **`Code.gs`** の中身を丸ごと貼り付け
4. 上部の **プロジェクト設定（⚙）→ タイムゾーン** が `(GMT+09:00) 東京` になっているか確認（日本ならこれでOK）
5. 保存（💾）

### CONFIG（コード冒頭）の調整 ※基本そのままでOK
| 項目 | 既定 | 説明 |
|---|---|---|
| `MONTHLY_SHEET` | `""` | 月次フォーマットのシート名。空＝先頭シート。名前が違う/複数シートあるなら `"勤務時間表"` 等を指定 |
| `DETAIL_SHEET` | `"明細"` | 明細ログのシート名（無ければ自動作成） |
| `WRITE_NIGHT` | `true` | 深夜労働列をアプリの正確な値で上書き。既存の式を残したいなら `false` |
| `WRITE_WORK` | `true` | 勤務時間・深夜休憩時間をアプリの値で入力。テンプレ側に式があるなら `false` で式を優先 |
| `CLEAR_EMPTY` | `true` | その月に記録の無い日の 出社/退社/休憩/勤務時間/深夜休憩 を空にする |

## 2. ウェブアプリとして公開

1. Apps Script 右上 **デプロイ → 新しいデプロイ**
2. 種類（⚙）で **ウェブアプリ** を選択
3. 設定：
   - 説明：任意（例 `勤怠連携`）
   - 次のユーザーとして実行：**自分**
   - アクセスできるユーザー：**全員**（※URLを知っている人だけが叩ける状態。データはあなたのシートにのみ書き込まれます）
4. **デプロイ** → 初回は権限承認（自分のGoogleアカウントを選び、「詳細」→「安全でないページに移動」→許可）
5. 表示される **ウェブアプリURL**（`https://script.google.com/macros/s/..../exec`）をコピー

> コードを直したら、**デプロイ → デプロイを管理 → 編集（鉛筆）→ バージョン「新バージョン」→ デプロイ** で更新します（URLは変わりません）。

---

## 3. アプリ側にURLを登録

1. 勤怠アプリを開く → **設定** タブ → 「スプレッドシート連携」
2. コピーした **ウェブアプリURL** を貼り付け → **URLを保存**
3. **シートに同期** を押す
   - 成功すると「✓ シート更新：月次 N日分（YYYY-MM）／明細 M件」と表示
   - 集計タブの「スプレッドシートに同期」ボタンからも実行できます

---

## 仕組み・注意点

- 月次シートは **シート上で選択中の年・月**（`B13` の月／年セル）の分だけ集計して書き込みます。別の月を出したいときは、シートの月を切り替えてから同期してください。
- アプリは「全セッション」を送り、シート側で当月分を抽出します。送信内容は **あなたのシートにだけ** 反映されます（外部に保存されません）。
- 月次の **休憩時間 = (退社 − 出社) − 実働** なので、休憩ボタンを押し忘れて勤務を中断しても、その空き時間は自動的に休憩として扱われます。
- CORS回避のため `Content-Type: text/plain` で送信しています（GASウェブアプリの仕様上これが必要）。
